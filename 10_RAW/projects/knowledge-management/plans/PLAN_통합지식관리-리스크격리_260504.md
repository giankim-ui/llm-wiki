## 핵심 원칙 (비개발자용 Git 멘탈 모델)

> **"모든 위험한 작업 전에는 ① 커밋(저장점) ② 새 브랜치(딴 우주) ③ 푸시(원격 백업) 3종 세트를 깐다."**

- **커밋(commit)**: 현재 폴더 상태의 스냅샷. 언제든 되돌아올 수 있는 저장점
- **브랜치(branch)**: 평행 우주. master는 절대 안 건드리고, 새 브랜치에서만 실험
- **푸시(push)**: GitHub에 백업. 로컬 PC 사고 나도 복구 가능
- **태그(tag)**: 특히 중요한 커밋에 라벨. `git checkout v-pre-migration` 으로 한방에 복귀
- **롤백 1줄**: `git checkout master` → 실험 통째로 무효화

비개발자는 이 5개 단어만 알면 충분. **Claude Code가 명령어를 만들어주고, 사용자는 "이거 실행해줘"만 승인**하는 흐름.

## 단계별 브랜치 전략

### Phase 0 — 안전망 구축 (30분, 코드 변경 0)
> **목적**: 어떤 마이그레이션도 시작하기 전에 "확실한 후퇴 지점" 만들기

1. 현재 master에 작업 중인 변경(`git status` M 표시 다수) 정리
   - 남길 변경 → `feat/pre-migration-cleanup` 브랜치에 커밋
   - 버릴 변경 → 사용자 확인 후 stash 또는 폐기 (절대 자동 폐기 금지)
2. **태그 생성**: `git tag v-pre-kb-migration` — "지식관리 마이그 시작 전 상태"
3. **원격 백업**: `git push origin master --tags` — GitHub에 안전 사본
4. 새 브랜치 생성: `git checkout -b kb/phase1-skeleton` — 모든 마이그 작업의 부모 브랜치

### Phase 1 — 골격 만들기 (원본 Week 1, 약 2시간)
- **브랜치**: `kb/phase1-skeleton`
- **작업**: `00_INBOX`, `10_ASSETS`, ..., `90_ARCHIVE` 빈 폴더 + `_templates/` + `MAP.md`

### Phase 2 — 자산 이관 (원본 Week 2, 약 4시간, **위험 중간**)
- **브랜치**: `kb/phase2-migration` (master에서 분기, phase1 머지 후)

- **위험**: 파일 이동 중 누락/덮어쓰기 가능
- **세이프티**:
  - `git mv` 사용 (히스토리 보존, 단순 cp + rm 금지)
  - **종목 1개씩 커밋** (예: "MSFT migration", "088980 migration") — 사고 시 종목 단위로 revert
  - 이관 후 원본 폴더(`S-anlyz/!Report/`)는 **삭제하지 말고 그대로 둠**. 양쪽 다 있는 상태로 1주 운영 후 검증되면 그때 삭제

> "Phase 2 끝났어. 검증 통과했으니 master에 머지해줘"

→ Claude Code가 머지 + 태그 + 푸시 명령 묶음 제시 → 사용자 승인 → 실행
