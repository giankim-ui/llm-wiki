'KEY RUSULTS' 의 행들과 TASK 내에 있는 행들은 계층구조가 동일한 범위 내에서 'Drag and Drop 기능' 추가가 필요해. 기존에 요청을 했는데 누락된것 같아.


1.
[기능 추가 요청] Drag & Drop - 행 순서 변경

📌 적용 위치
1)OKR 섹션의 KEY RESULTS 목록
2)하단 TASK 섹션 목록에서의 TASK 및 SUB-TASK  목록

📌 동작 규칙
1. OKR 섹션과 TASK 섹션 간은 분리되어 있으므로 상호 이동되지 않는다.
3. TASK 섹션 내 TASK 행은 같은 KEY RESULT 안에 있는 TASK끼리만 순서 변경 가능

📌 UI 동작
- 드래그 중: 해당 행이 반투명하게 표시되고, 놓을 위치에 파란 선(인디케이터) 표시
- 드롭 후: 변경된 순서가 즉시 화면에 반영되고, 서버(DB)에 저장

📌 드래그 핸들
- 각 행 왼쪽에 ⠿ (핸들 아이콘) 추가, 해당 아이콘을 잡아야만 드래그 가능


'TASK 섹션'에서 'sub-task'를 수정하여 EM의 순서를 1->3으로 변경했을 때 TASK->SUB-TASK 간의 계층구조가 깨져서 이미지와 같이 다른 'TASK'서 보여지고 있어
같은 'TASK' 내의 값인 '시큐어 내에 기존 자료 리스트화' 아래 계층에 있는 'SUB-TASK' 값인 '인원현황 명부 10개년 정리'와  'TEST'는 같이 묶여 있어야 해.



2.
🐛 버그 설명
TASK 섹션에서 SUB-TASK의 [EM 필드 / 정렬순서 필드] 값을 1에서 3으로 변경했더니,
SUB-TASK가 원래 부모 TASK에서 벗어나 다른 TASK 아래에 표시되는 문제가 발생했습니다.

📍 재현 방법
1. TASK: '시큐어 내에 기존 자료 리스트화' 항목 중 그 하위의 SUB-TASK인 '인원현황 명부 10개년 정리'의 수정 버튼을 클릭합니다.
2. EM 값을 1 → 3으로 수정합니다.
3. 저장 후 화면을 확인합니다.

❌ 현재 잘못된 동작
'인원현황 명부 10개년 정리'와 'TEST' SUB-TASK가
'시큐어 내에 기존 자료 리스트화'가 아닌 다른 TASK 아래에 표시됨.

✅ 기대하는 동작
EM 값이 변경되어도, TASK와 SUB-TASK의 부모-자식 관계(계층 구조)는
유지되어야 합니다.
'인원현황 명부 10개년 정리'와 'TEST'는 항상
'시큐어 내에 기존 자료 리스트화' TASK 아래에 묶여 있어야 합니다.

🔗 관계 구조 (현재 이해한 것)
TASK: 시큐어 내에 기존 자료 리스트화
  └─ SUB-TASK: 인원현황 명부 10개년 정리  (EM: 3으로 변경)
  └─ SUB-TASK: TEST

📎 [스크린샷 첨부 - 깨진 화면]

[수정 요청] OKR 섹션 아코디언 UI 개선


##현재 상태:

화면 상단에 OKR 섹션(헤더 포함)이 position: sticky 또는 고정(fixed) 처리되어 있음
KR NO., KEY RESULTS 목록이 항상 펼쳐진 상태로 표시되어 아래 콘텐츠 영역이 좁아지는 문제


##원하는 변경 사항:

OKR 섹션 헤더 행 (KR NO. / KEY RESULTS 컬럼 헤더가 있는 행) 의 가장 오른쪽 끝에 토글 버튼 추가
버튼 모양: 아래 방향 삼각형 ▽, 색상은 주황색 (#FF6600 계열)
버튼 클릭 시 **KR 데이터 행 전체(헤더 아래 목록)**가 접힘(collapse) → 아코디언 방식
다시 클릭하면 펼쳐짐(expand), 이때 버튼은 △로 전환
기본 상태는 **펼쳐진 상태(expanded)**로 유지
헤더 행 자체의 고정(sticky)은 유지할 것
