# DuckDB 기반 HR 대시보드 구축 검토 요약

> 작성일: 2026-05-11  
> 목적: plan.md 반영용 — DuckDB + Python + HTML 대시보드 아키텍처 검토 결과

---

## 1. 배경 및 목적

- 분기별로 작성된 엑셀 파일(인원, 인건비, 생산성 데이터)을 효율적으로 저장·조회
- Python 기반 구조로 HTML 대시보드에 데이터를 연결
- 수정은 담당자 1인(내 PC), 검토는 상위자(다른 PC) 구조

---

## 2. 핵심 질문 및 결론

### Q1. 엑셀 전체를 Claude에게 주는 것과 DuckDB 방식의 차이는?

| 항목 | 엑셀 → Claude | DuckDB + Python |
|---|---|---|
| 데이터 용량 | 1GB 업로드 불가 | 제한 없음 |
| 영속성 | 대화 종료 시 소멸 | 로컬 파일로 영구 보존 |
| 대시보드 연동 | 불가 | 실시간 쿼리 연동 |
| 보안 | 외부 전송 | 로컬/사내망 한정 |

**결론:** DuckDB 방식이 구조적으로 우월. Claude는 코드를 생성하고, 데이터는 로컬에 보존.

---

### Q2. 스키마 확정 후 열(컬럼) 추가가 복잡한가?

**전혀 복잡하지 않음.** 한 줄로 처리 가능.

```sql
-- 열 추가
ALTER TABLE hr_data ADD COLUMN 외국인_비율 DOUBLE;

-- 기존 데이터에 계산값 일괄 적용
ALTER TABLE hr_data ADD COLUMN 인당_인건비 DOUBLE;
UPDATE hr_data SET 인당_인건비 = 인건비총액 / 인원수;
```

**주의사항:** 스키마 변경 자체보다 초기 **데이터 타입 설계**가 더 중요. (예: 분기 표기를 텍스트 vs 날짜형으로 정의하는 것)

---

### Q3. DuckDB vs Supabase — 어느 용량까지 DuckDB가 유리한가?

| 용량 | DuckDB | Supabase |
|---|---|---|
| ~50GB | ✅ 유리 | 오버스펙 |
| 50~100GB | 상황에 따라 | 유리해지기 시작 |
| 100GB+ | 한계 체감 | ✅ 유리 |

**실제 갈림길은 용량이 아닌 3가지 요소:**

1. **동시 접속 여부** — DuckDB는 단일 프로세스, 동시 쓰기 불가
2. **데이터 갱신 주기** — 월/분기 단위라면 DuckDB로 충분
3. **보안 정책** — 인사 데이터의 외부 클라우드 적재 이슈 → DuckDB(로컬)가 안전

**결론:** Pulmuone HR 데이터(분기 갱신, 소수 담당자, 보안 민감) 용도에는 **DuckDB가 적합**. Supabase는 전 직원 셀프서비스 포털 수준으로 확장 시 검토.

### Q4. 다른 PC에서 HTML을 보려면 DuckDB가 설치되어야 하나?

**맞음.** DuckDB + Python이 없는 PC에서는 동작하지 않음.

**해결 방안 2가지:**

#### ✅ 방안 A — 데이터를 HTML에 구워넣기 (최종 배포용)
- Python 1회 실행 → 데이터가 JSON으로 HTML 내부에 포함된 완성 파일 생성
- 받는 사람은 HTML 파일만 있으면 브라우저에서 바로 열람 가능
- **단점:** 데이터 변경 시 Python 재실행 후 HTML 재배포 필요

#### ✅ 방안 B — 로컬 서버 공유 (개발/검토 과정용)
- 내 PC에서 아래 명령어 실행:
  ```bash
  python -m http.server 8000
  ```
- 사내망 내 다른 PC에서 브라우저로 접속:
  ```
  http://[내PC_IP주소]:8000/dashboard.html
  ```
- 내가 수정 → 저장 → 상위자가 새로고침만 하면 반영
- 추가 설치 없음 (Python만 있으면 됨)
- **단점:** 내 PC가 켜져 있어야 접속 가능

## 3. 최종 채택 아키텍처

```
[원천 데이터]         [저장소]           [처리]         [표현]
엑셀 (분기별) ──▶ DuckDB (.duckdb) ──▶ Python ──▶ HTML 대시보드
```

### 단계별 운영 방식

| 단계 | 방식 |
|---|---|
| **개발 · 검토 중** | `python -m http.server`로 사내망 공유 → 상위자 브라우저 확인 |
| **최종 배포** | 데이터 HTML에 구워서 파일 배포 (이메일 등) |
| **데이터 갱신** | 분기마다 Python 1회 실행 → HTML 재생성 |

## 4. 설치 필요 항목 (담당자 PC 기준)

```bash
# Python 설치 후 (python.org, 3.8 이상)
pip install duckdb pandas openpyxl
```

| 패키지 | 역할 |
|---|---|
| `duckdb` | DB 본체 |
| `pandas` | 쿼리 결과 → 데이터 변환 |
| `openpyxl` | 엑셀 파일 읽기 |

> 열람자 PC에는 별도 설치 불필요 (브라우저만 있으면 됨)
