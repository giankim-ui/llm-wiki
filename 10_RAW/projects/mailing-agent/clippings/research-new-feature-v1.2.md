# 현재 mailing-agent를 개선하려고해.

## 개발 우선순위
```
1순위: Item 3 (질문 단순화)   — draft.md 수정만으로 즉시 효과, 리스크 없음
2순위: Item 2 (사용자 등록)   — DB 구조 변경 포함, 중간 난이도
3순위: Item 1 (아웃룩 연동)   — COM Automation 경로로 진행, IT 승인 불필요
4순위: Item 4 (표 기능)       — Item 1 완료 후 pywin32 재활용으로 구현
```

## 1. 초안 아웃룩 임시보관함 자동 이동 기능

- 내 MS 아웃룩 환경은 M365야. 현재 mailing-agent가 작업한 결과물 초안을 자동으로 내 아웃룩 내 임시보관함에 저장하는 기능을 추가해줘.
- 해당 내용을 첫 개발로 하고 phase를 구분해줘.
- 코딩 후 반드시 시험 작동 여부를 먼저 사용자에게 테스트 해줘.

### 구현 방향

**COM Automation (pywin32) 방식으로 개발** — Azure AD 앱 등록 불필요, IT 부서 승인 없이 즉시 개발 가능.

| 방법 | 특징 | 이슈 |
|------|------|------|
| **COM Automation** (채택) | pywin32로 로컬 Outlook 직접 제어 | Outlook 앱이 실행 중이어야 동작 |
| Microsoft Graph API | 안정적, 앱 실행 여부 무관 | 회사 Azure AD 앱 등록 필요 → IT 부서 승인 필요 가능성 |

> Graph API는 IT 정책 승인이 변수가 될 수 있어 후순위 대안으로 남겨둠.
> pywin32는 Item 4 표 기능(클립보드)에서도 재사용 가능.

### Phase 구분
- **Phase 1**: pywin32 환경 구성 및 Outlook 연결 확인
- **Phase 2**: 초안 내용을 Outlook 임시보관함에 저장하는 기능 구현
- **Phase 3**: 사용자 테스트 및 오류 처리


## 2. 최초 사용자 등록

- 본 에이전트를 여러 명이 사용할 것을 가정하여 최초 1회에 한하여 사용자의 '성명' '조직명'을 입력하는 프로세스를 추가해줘.
- 이미 '성명' '조직명'이 있을 경우에는 생략해.
- 이미 개발된 내용이라면 생략해도 돼.

### 구현 방향

**`data/emails.db`에 `users` 테이블 추가** — 기존 SQLite를 그대로 활용, 추가 의존성 없음.

- CLAUDE.md에 저장하지 않는 이유: CLAUDE.md는 프로젝트 공유 파일이므로 다중 사용자가 쓸 경우 덮어쓰기 충돌 발생
- CLAUDE.md에는 "현재 활성 사용자: `{이름}`" 참조만 기록
- 현재 CLAUDE.md에 하드코딩된 `김지형, 인사혁신팀`을 DB 기반으로 동적 교체하는 작업 포함


## 3. /draft 이후 질문 단순화

`commands/draft.md` 수정 내용 분석을 기반으로 아래와 같이 반영한다.

### 3-1. Step 1 자동화 (메일 유형 선택 제거)
- **기존**: 사용자가 6개 유형 중 하나를 수동 선택
- **변경**: Step 2에서 입력받은 핵심 정보(목적 및 수신자, 주요내용)를 분석하여 유형을 자동 판단
- **구현 방향**: draft-writer 에이전트가 핵심 정보를 바탕으로 메일 유형을 추론 후 초안 생성에 활용
- **추가**: 추론 결과를 초안 상단에 명시 (예: "결재 요청 메일로 판단하여 작성했습니다") → 사용자가 오판 여부 즉시 확인 가능

### 3-2. Step 2 입력 필드 구조 개편
기존 6개 필드를 3개로 간소화한다.

| 기존 필드 | 처리 |
|----------|------|
| 수신자 | 목적 필드와 통합 |
| 참조 | 삭제 |
| 목적 | 수신자 포함 형태로 통합 (예: "담당님께 Claude Team Plan 구매 사전 결재") |
| 주요 내용 | 기한/일정과 통합 → "주요내용 및 기한/일정" 단일 필드 |
| 기한/일정 | 주요 내용에 통합 |
| 첨부파일 | 삭제 |

**변경 후 입력 필드 (3개):**
```
다음 정보를 입력해주세요. (해당 없는 항목은 'n' 입력)

목적 및 수신자: (예: 담당님께 Claude Team Plan 구매 사전 결재)
주요내용 및 기한: (핵심 사실/수치, 기한이 있다면 함께 기재)
참고 메일: (data/emails/파일명.eml - 없으면 'n')
```

> 주의: `첨부파일` 필드를 삭제하더라도 CLAUDE.md의 "첨부파일 언급 시 알림 필수" 규칙은 유지.
> 사용자가 `주요내용` 필드에 첨부파일을 언급하면 draft-writer가 자동 감지하는 로직 필요.

### 3-3. Enter 건너뛰기 버그 수정
- **근본 원인**: Claude Code CLI는 빈 메시지(Enter만)를 전송하지 않는 구조적 제약 → "Enter로 건너뛰기" 패턴 자체가 이 환경에서 작동하지 않음
- **처리**: 안내 문구를 "없으면 'n' 입력"으로 변경 (3-2 개편 작업과 동시 처리)


## 4. 표 기능 개선

아웃룩은 HTML 붙여넣기를 미지원하나 실무에서는 마크다운 형태의 표는 사용하지 않아. 표 형식 정리를 지원할 수 있는 방법을 research 단계로 진행한다.

### 구현 방향

**Python + pywin32 방식** — npm/Node.js 불필요, 기존 Python 스택 유지.

| 경로 | 패키지 | 비고 |
|------|--------|------|
| **Python** (채택) | `pywin32` (win32clipboard) + `markdown` | Item 1 COM Automation과 의존성 공유 |
| npm | `marked` + `clipboardy` | Node.js 런타임 별도 필요, 기존 스택과 이질적 |

**동작 방식**: 마크다운 표 → HTML 변환 → Windows 클립보드에 HTML 형식 저장 → Outlook Ctrl+V 붙여넣기 시 표 서식 유지

> Item 1 개발 시 pywin32를 먼저 도입하면 이 기능 구현 비용이 크게 낮아짐.
